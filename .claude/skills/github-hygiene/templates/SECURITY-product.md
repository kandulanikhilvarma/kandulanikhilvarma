# Security Policy

## Reporting a Vulnerability

If you discover a security or privacy vulnerability in this project, please
report it privately rather than opening a public issue:

- **Email:** kandulanikhilvarma@gmail.com (subject line: `SECURITY`)
- Please **do not** open a public issue, pull request, or discussion for a
  security report, and please do not disclose the issue publicly until it has
  been resolved.

Where possible, include a description of the issue, steps to reproduce, the
affected component, and any relevant logs. I aim to acknowledge reports within
**72 hours** and to share a remediation timeline after an initial assessment.

## Supported Versions

Only the latest state of the default branch (`main`) is supported. Security
fixes are applied to `main` and deployed forward; older commits and tags are not
patched.

## Good-Faith Research

Good-faith security research is welcome. While testing, please avoid privacy
violations, data exfiltration, service disruption, and accessing data that is
not yours.
